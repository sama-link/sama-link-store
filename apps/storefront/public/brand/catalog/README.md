# Brand catalog logos

Public path: `/brand/catalog/<handle>.webp`
Convention: 512×512, transparent background, centered, ≥10% padding.

Each `Brand.image_url` in the backend can be set to either:

- a **relative path** like `/brand/catalog/tenda.webp` — served from this folder by the storefront. Works in the storefront browser; the admin preview uses `NEXT_PUBLIC_STOREFRONT_BASE_URL` (or the configured admin shim) to resolve it.
- an **absolute URL** to the brand's official CDN — works from any browser.

## How to replace a placeholder logo

1. Drop a real `<handle>.webp` into this folder using the brand's lowercased handle (matches `Brand.handle` in DB).
2. Keep the file 512×512 with transparent background and balanced padding.
3. No DB change required — files load by URL.

## Regenerating the demo placeholder logos

The current set was generated by `scripts/generate-brand-catalog-logos.mjs` (in the repo root) using `sharp`. Re-run it from the repo root to refresh.

These are wordmark placeholders — they show the brand name in an approximated brand color, NOT the official brand mark. Replace them with the official `.webp` per brand before any production launch.
